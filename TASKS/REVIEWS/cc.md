1. Корневая причина не обязательно в calculate_wind_components
MiMo теперь пишет:
Crosswind sign инвертирован. Формула должна давать отрицательное значение для ветра справа.
Это не совсем точно относительно текущего контракта.
Docstring явно определяет:
crosswind > 0 = ветер справа
crosswind < 0 = ветер слева
​
При таком определении:
wind from 90°, heading 0° → crosswind = +20
​
является корректным результатом: ветер действительно приходит справа.
crosswind здесь обозначает сторону, откуда приходит ветер, а не поперечную компоненту физического вектора скорости воздуха.
Ошибка возникает дальше: код получает «ветер справа», но направляет нос влево:
if crosswind > 0:
    crab = -crab
​
При сохранении текущего контракта правильная логика:
ratio = max(-1.0, min(1.0, crosswind / true_airspeed))
crab = math.degrees(math.asin(ratio))
corrected_heading = (desired_track + crab) % 360
​
То есть:
ветер справа, crosswind > 0 → положительный crab → нос вправо;
ветер слева, crosswind < 0 → отрицательный crab → нос влево.
Альтернатива — инвертировать знак в calculate_wind_components, но тогда потребуется одновременно изменить:
docstring;
calculate_drift_angle;
calculate_crab_angle;
всех потребителей signed crosswind;
тесты и логи.
Минимальный безопасный фикс — сохранить documented crosswind convention и исправить crab, а не менять базовый контракт компонента.
